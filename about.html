Data Model

Database Schema

Our application uses a normalized relational database design with the following key relationships:

Users to Bookings (1:N)

A user can make multiple bookings, but each booking belongs to one user.

Destinations to Bookings (1:N)

A destination can be included in multiple bookings, but each booking is for one destination.

Bookings to Review (1:N)

A booking can have multiple reviews (for different aspects), but each review belongs to one booking.
